New additions to the Cinema 4D Python SDK are marked like this throughout the documentation:

New in version R19.053.

Changes¶

  • The following functions now return a c4d.GeListHead object:
    • c4d.AllocListHead()
    • GeListNode.GetListHead()
    • BaseList2D.GetCTrackRoot() and BaseList2D.GetNLARoot()

  • The following functions now accept or return a c4d.documents.BaseVideoPost:
    • RenderData.GetFirstVideoPost()/RenderData.InsertVideoPost()/RenderData.InsertVideoPostLast()

  • The following c4d.modules.motiontracker types no longer use Sdk suffix:
    • FootageDataSdk renamed as MtFootageData
    • Mt2dTrackDataSdk renamed as Mt2dTrackData
    • Mt2dTrackSdk renamed as Mt2dTrack
    • MtDataSdk renamed as MtData
    • MtTrkGidSdk renamed as MtTrkGid

Parameters¶

  • New optional parameter dirtyflags for BaseObject.GetHierarchyClone() and BaseObject.AddDependence()
  • New optional parameter col for c4d.gui.ColorDialog() to initialize default color

Miscellaneous¶

  • PolygonObject.GetPolygonTranslationMap() now returns a tuple with the number of N-gons and the list of N-gon indices

R19.053¶

  • c4d.gui.ActiveObjectManager_SetObjects()

  • Description.GetParameter()/GetParameterI() parameter ar is optional
  • SoundEffectorData.Sample() returns None values if it fails
  • CalcVertexMap() name should be used instead of old naming PointObject.CalcVertexmap()

R19.068¶

  • New optional parameter visible for c4d.utils.CalculateVisiblePoints()
